## 登入

連線到 DOMjudge 主畫面後，請點擊 `Login` 的登入按鈕。

![login-0](https://hackmd.io/_uploads/ByAhhAveGl.jpg)

在 login 的登入畫面：

1. 在 `username` 的位置輸入你的帳號
2. 在 `password` 的位置輸入你的密碼
3. 點擊 `sign in` 進行登入

![login-1](https://hackmd.io/_uploads/S1RHTRvxMg.jpg)

登入後主畫面功能說明：

1. Home 是 DOMjudge 個人積分板的首頁，以及當前考場的解題狀況
2. Problemset 指的是在當前考場中所有的題目，這個頁面可以下載測資跟題目 PDF 檔
3. Scoreboard 是全體的積分板狀態；當凍結時，積分板其他的解析狀態會變成固定 Pending 的顏色
4. Submit 是你提交解題程式碼的按鈕
5. Logout 是登出
6. 預設剛登入會顯示 `no contest`，代表目前還沒有生效任何考場可以進行解題
7. 會顯示帳號資訊，這是登入成功的畫面

![login-2](https://hackmd.io/_uploads/Hk-UAADeze.jpg)

## Home：個人積分板

1. 上方功能選單的 Home 是顯示個人積分板的頁面連結
2. 當前考場的倒數時間
3. 當前考場個人的積分板、解題的進度
4. 是你當前提交解題的記錄跟解題的結果狀態
5. 裁判回覆問題的訊息，或者是全體公告訊息的列表
6. 當前使用者之前提問的記錄
7. 新增提問的按鈕

![home-0](https://hackmd.io/_uploads/BkNzHgdgfx.jpg)

### Submissions

送出提交的程式碼的紀錄。

#### result（伺服器評判結果）

- `CORRECT`（正確答案）：提交的程式碼，伺服器評判成功。
- `COMPILER-ERROR`（編譯錯誤）：程式碼無法成功編譯。
- `TIMELIMIT`（超過時間限制 / TLE）：提交的程式碼執行耗時過長或是陷入無限迴圈，顯示超過時間題目限制。
- `RUN-ERROR`（執行時期錯誤 / RE）：提交的程式結束時非以結束碼（Exit Code）0 正常結束，這可能由多種原因引起，例如：除以零、記憶體存取錯誤（例如：陣列索引越界）、意圖使用超過限制的記憶體、嘗試進行檔案讀寫等。
- `WRONG-ANSWER`（答案錯誤 / WA）：伺服器執行你所提交程式輸出的結果與答案不匹配，即為執行結果錯誤。
- `NO-OUTPUT`（無輸出）：伺服器執行你所提交程式無法取得輸出結果。請檢查程式邏輯是否確實將結果寫入到標準輸出（Standard Output, stdout）。
- `OUTPUT-LIMIT`（超過輸出限制）：程式產生的輸出資料量超過了允許的上限。此狀態會被視為解答錯誤。
- `TOO-LATE`（錯過提交時間）：競賽結束後才提交，程式碼已被系統儲存，但將不會再進行任何評測。

## Problemset：當前考場的題組

1. Problemset：功能選單中可連結到考場題組的功能
2. 每一個卡片都是一個題目
3. Statement：點擊後可以看到題目的 PDF
4. Samples：可以下載範例的輸入測資以及輸出的答案，有些題目可能會沒有這個選項
5. Submit：用於上傳提交的程式碼

![problemset-0](https://hackmd.io/_uploads/SJ_PgW_xfe.jpg)

## Submit：上傳提交的程式碼

在任意的頁面有 Submit 的按鈕，都可以點擊進行提交程式碼。

![submit](https://hackmd.io/_uploads/rkumQbdgGx.jpg)

1. 請選擇你要上傳提交的程式碼檔案，檔名盡量與你要提交的題目的編號相同。例如題目是 L04，你可以將你的檔案命名為 L04.py（.py 是指下面要選擇上傳 Python 檔案的意思）。
2. Problem：選擇你要上傳的題目。
3. Language：選擇你要上傳所執行的程式語言。
4. Submit：提交上傳的程式碼。

![submit-1](https://hackmd.io/_uploads/SylxNZuefe.jpg)

## Scoreboard：當前考場全體積分板

![scoreboard-0](https://hackmd.io/_uploads/HJOUA-dxzx.jpg)

### 1. Solved first（首殺 / 最快解出）

- **顏色：** 深綠色
- **定義：** 這是該道題目在**整個競賽全場中「第一個」被成功解出**的隊伍才會獲得的特殊榮譽。
- **畫面呈現：** 該格子除了會顯示解出時間（從開賽算起的分鐘數）與提交次數外，顏色會比一般的「Solved」更深。在 ICPC 傳統中，這通常也代表該隊伍可以獲得該題目的「首殺氣球（First to solve balloon）」。

### 2. Solved（正確解答 / 通過）

- **顏色：** 淺綠色
- **定義：** 代表隊伍已經**成功通過該題目的所有測試資料**（獲得 `CORRECT` 狀態），成功解決了這道題。
- **畫面呈現：** 欄位內會標記答對當下的比賽時間（分鐘）以及答對前（含答對那次）的總提交次數（例如 `1 try` 或 `3 tries`）。只要欄位變綠，該題不論之前失敗幾次，自此之後都不會再因為這題累積新的罰時。

### 3. Tried, incorrect（曾嘗試但答案錯誤）

- **顏色：** 紅色
- **定義：** 代表隊伍**曾經提交過這道題目的程式碼，但評測結果全數失敗**（例如獲得 `WRONG-ANSWER`、`TIMELIMIT`、`RUN-ERROR` 等非正確 Verdict），且**目前尚未解出**。
- **畫面呈現：** 欄位內會顯示目前為止該題的失敗提交次數。這提醒隊伍這題已經投資了時間與嘗試次數，若之後成功解出，這些失敗的次數將會被納入罰時（Penalty time）計算（預設每次錯誤提交加罰 20 分鐘）。

### 4. Tried, pending（已嘗試，評判中 / 待定）

- **顏色：** 藍色
- **定義：** 代表隊伍**剛剛提交了這道題目的程式碼，但系統目前還在進行評測中（Pending）**，或者該場競賽已經進入了「封榜（Scoreboard Freeze）」階段。
- **設計理念說明（封榜機制）：** 管理員會開啟封榜功能，或是稱為凍結計分版功能。開啟後，全體計分板將不再更新最終結果。如果某個隊伍在封榜後提交，除了隊伍自己的主頁，公開計分板上該題的格子就會變成這個**藍色**，代表「有丟程式碼，但大家還不知道他到底有沒有過」。

### 5. Untried（未嘗試）

- **顏色：** 白色 ⬜
- **定義：** 代表該隊伍**尚未對這道題目提交過任何程式碼**。這是比賽開始時所有題目的預設初始狀態。

## Clarifications：當前考場問題提問

所有與裁判之間的溝通，都必須透過「問題提問（Clarification messages）」來進行。您可以在團隊主頁的右側欄位中找到此功能，不論是裁判回覆的說明，或是您所發送的請求，都會顯示在該處。

頁面上也有一個按鈕，可用來向裁判提交新的問題提問；您可以將請求關聯至特定的「題目」或是「通用類別」。這項問題提問只有裁判能夠閱讀。裁判可以單獨回覆給您，如果該問題與所有參賽者皆有關係，裁判也可以將回覆發送給全場所有人。

![clarification-0](https://hackmd.io/_uploads/H1k0VM_xfg.jpg)

- Recipient（收件者）：此欄位為預設值 `Jury`（裁判團 / 評審團），代表您這則訊息是**直接發送給考場的裁判**。在 DOMjudge 的系統設計理念中，一般隊伍無法透過此功能直接傳送訊息給其他參賽隊伍，所有提問皆是由裁判統一接收與處理。
- Subject（主題 / 關聯題目）：這是一個下拉式選單。您可以將這個提問**綁定到目前正在嘗試的具體題目**（如此處選擇的 `L04`），這樣裁判在審閱時能立刻明白您是在詢問哪一道題。如果是與特定題目無關的通用問題（例如系統異常、座位環境等），選單中通常也可以選擇「General（一般類別）」。
- Message（提問內容輸入框）：填寫具體提問內容的地方。下方還帶有 `rendered preview`（預覽區塊），可以即時顯示您輸入的文字。在撰寫此區塊時，建議保持簡潔、明確，並指出具體發現問題的測資狀況，以便裁判快速判斷。
- Send：點擊 **`Send`** 按鈕後，這則提問就會正式發送給裁判。發送後，您可以隨時在團隊主頁右側的說明欄位中，查看這則訊息的狀態以及裁判後續的回覆。

![clarification-1](https://hackmd.io/_uploads/Bk003GOxzg.jpg)

- reply to this clarification（回覆此說明）：如果裁判回覆您的提問後，您發現問題仍未解決，或者您想要針對同一個問題繼續補充說明，可以點擊這個按鈕。這樣可以讓後續的討論綁定在同一個對話串（Thread）中，方便裁判追蹤前後文，而不需要重新建立一個全新的提問。
